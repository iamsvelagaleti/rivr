011/**
012 * Generates part of the VoiceXML document responsible for handling of VoiceXML
013 * fatal errors (error events that can't be handle by the HTTP server).
014 * <p>
015 * Normally, VoiceXML <code>error</code> events are sent back to the dialogue
016 * for processing. However, if another error occurs when the first error is sent
017 * back to the server, we have a case of fatal error and must have an error
018 * handler in the VoiceXML to deal with these errors. This class contains the
019 * {@link #addFatalErrorForm(Element, VoiceXmlDocumentTurn)} method which is
020 * invoked to add the proper VoiceXML elements in order to deal properly with
021 * such errors.
022 * <p>
023 * By default, the {@link VoiceXmlDialogueServlet} is initialized with a
024 * {@link ExitFatalErrorFormFactory}, which generates an empty
025 * <code>&lt;exit&gt;</code> element. This is generally acceptable. However,
026 * some contexts would require a different handler, such as subdialogues where a
027 * <code>&lt;return&gt;</code> would be more appropriate.
028 * 
029 * @see ExitFatalErrorFormFactory
030 * @see ReturnFatalErrorFormFactory
031 * @author Nu Echo Inc.
032 */
033public interface FatalErrorFormFactory {
034    void addFatalErrorForm(Element form, VoiceXmlDocumentTurn turn);
035}
