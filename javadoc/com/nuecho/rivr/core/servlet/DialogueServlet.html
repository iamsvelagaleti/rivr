    public abstract class DialogueServlet<I extends InputTurn,O extends OutputTurn,F extends FirstTurn,L extends LastTurn,C extends DialogueContext<I,O>>
extends HttpServlet
    Abstract servlet interacting with a web client acting as the controller of a SynchronousDialogueChannel.

    This abstract servlet must be extended in order to provide a specific implementation. For each session,

    1. the servlet creates the Session and place it in the SessionContainer
    2. it creates a Dialogue with a DialogueFactory
    3. it creates a DialogueContext with a DialogueContextFactory
    4. it creates a SynchronousDialogueChannel and starts the dialogue upon initial HTTP request
    5. it renders the various steps from the dialogue channel into appropriate HTTP responses
    6. it translates HTTP requests into InputTurns using the InputTurnFactory
    7. once the dialogue is done, the servlet perform necessary clean-up.

    init args

    The following servlet initial arguments are supported:

    com.nuecho.rivr.core.dialogueTimeout
    Maximum time for dialogue to produce an OutputTurn. Value specified must be followed by unit (ms, s, m, h, d, y), e.g. 10s for 10 seconds. Default value: 10 s
    com.nuecho.rivr.core.controllerTimeout
    Maximum time for controller to produce an InputTurn. Value specified must be followed by unit (ms, s, m, h, d, y), e.g. 10s for 10 seconds. Default value: 5 m
    com.nuecho.rivr.core.sessionTimeout
    Maximum inactivity time for a session. Value specified must be followed by unit (ms, s, m, h, d, y), e.g. 10s for 10 seconds. Default value: 30 m
    com.nuecho.rivr.core.sessionScanPeriod
    Time between each scan for dead sessions in the session container. Value specified must be followed by unit (ms, s, m, h, d, y), e.g. 10s for 10 seconds. Default value: 2 m
    com.nuecho.rivr.core.webappServerSessionTrackingEnabled
    Whether a HttpSession should be created for each dialogue or not. This is useful for load-balancers using JSESSIONID cookie to enforce server affinity (or stickyness). Value should be true or false. Default value: true

      • setWebappServerSessionTrackingEnabled

        public final void setWebappServerSessionTrackingEnabled(boolean enableWebappServerSessionTracking)
        Indicates if the servlet should create an HttpSession object for each dialogue. Note: Nothing is stored in the HttpSession. However, the creation of a session would force the web container to track the session using a cookie (JSESSIONID) or to do URL-rewriting. This is only relevant if there is more than one web container fronted by a load balancer.
        Parameters:
        enableWebappServerSessionTracking - true if HttpSession are to be used for session tracking.
        Since:
        1.0.1
